# AIGC Visual-Style Benchmark Matrix

Use this file to manage visual-style references separately from MG or other structure-led benchmark systems.

The goal is to keep style references useful without letting them overwrite film type, step structure, or production logic.

## How To Use This Matrix

For each style-led reference:

1. identify the dominant visual lane
2. decide whether it is only a surface-style benchmark or also a structural benchmark
3. transfer atmosphere, motif, typography, palette, or composition only when appropriate
4. keep `video_type` decisions anchored in the project's communication goal

Do not let a visual-style reference silently redefine the project type.

## Selection Rule For New Benchmarks

A new visual-style benchmark is worth adding only if it contributes a genuinely new visual lane.

Do add it when:

- it teaches a different atmosphere system
- it changes title-treatment thinking
- it shows a different image-composition logic
- it suggests a new kind of teaser or visual-promo energy

Do not add it when:

- it is just another surface variation of an existing lane
- it repeats the same hero-poster logic with a different costume or color
- it has no reusable lesson beyond "looks cool"

## Separation Rule

Always ask:

- does this reference teach structure
- or does it only teach style

If it only teaches style:

- keep it in this matrix
- do not move it into the MG benchmark system
- do not let it override step-specific planning logic

## Good Matrix Standard

A healthy visual-style matrix should let the system answer:

- what kind of atmosphere does the user want
- what kind of title or poster logic fits that atmosphere
- whether the reference affects only surface style or also short-form structure
- how to combine a style reference with a separate type benchmark without confusion
