# BC AIGC Motion Case

Use this file as a visual-style calibration case, not as a structure template.

This reference captures one short AIGC-driven kinetic motion promo pattern from the local benchmark video `BC.mp4`.

## Why This Case Matters

This case is useful because it is:

- clearly motion-led rather than information-led
- object-led rather than character-led
- a strong example of kinetic camera energy and continuous chase-style movement
- different from both MG explainers and poster-motion teaser references

Use it when the user wants:

- an AIGC motion-energy benchmark
- a fast object-follow or camera-chase reference
- a short promo where movement is the main attraction
- a reference for spatial travel across landmarks or environments

Do not use it to define explainer structure, MG planning, or workflow logic.

## Core Case Summary

Working description:

- a short-form AIGC motion promo
- a bright tennis-ball-like object travels through Paris landmarks and other spaces
- the core attraction is dynamic tracking, speed, direction changes, and spatial continuity
- the piece behaves like a camera-motion showcase more than a message-led short film

Approximate role:

1. establish the moving object and motion rule
2. accelerate through recognizable landmarks or spaces
3. sustain speed and spatial spectacle
4. close as a compact movement-driven promo beat

## Dominant Type Judgment

Primary type:

- `Trailer / Teaser Style Short`

Secondary supporting traits:

- `Concept Narrative Short`
- `AIGC Motion Showcase`
- `Object-Led Visual Promo`

Not primary type:

- `MG Explainer / Information Short`
- `Poster-Motion / Key-Art Animation`
- `UI Demo / Screenflow Video`
- `Data Visualization Film`

## Why It Is Not MG

This case is not a good MG benchmark because:

- it does not explain a concept through information hierarchy
- it is driven by kinetic camera experience rather than modular communication
- the moving object is the hook, not a system or workflow
- the value comes from motion spectacle, not from explainer clarity

## What This Case Is Good For

Transferable style signals in this case:

- one clear moving object as the visual anchor
- aggressive forward momentum
- chase-like camera behavior
- motion continuity across different spaces
- compact high-energy progression
- spatial spectacle through travel rather than through titles

## What Is Specific To This Case

These are case-specific decisions, not universal motion-style rules:

- Paris landmarks and European architecture
- tennis-ball-like neon object
- travel through iconic public spaces
- sports-object association
- `Metaverse AIGC` showcase framing

Do not generalize these details to every motion-driven visual promo.

## What Must Not Be Overgeneralized

Do not conclude from this case that:

- every motion-led AIGC promo needs a single ball object
- every spatial-chase piece needs famous landmarks
- all kinetic visual promos should be tourism-like
- camera-speed references can replace structure references

## Reusable Guardrails

These lessons transfer well to future motion-led AIGC promos:

- choose one dominant moving anchor
- keep motion readability high even when speed is strong
- make location transitions feel causally linked by movement
- let the object drive the edit rhythm
- keep the piece compact and momentum-first

## Reusable Freshness Levers

These are safe places to vary future motion-driven promos:

- object type
- camera-follow grammar
- environment family
- travel path logic
- speed profile
- collision or transition mechanic
- reveal pattern
- end-beat framing

## Step 1 Calibration

What this case teaches Step 1:

- some references mainly influence movement language, not format
- movement-first references belong to the visual-direction layer unless the final project is also a teaser or promo
- kinetic energy can be borrowed without inheriting the short's whole type

What Step 1 should not do:

- relabel an explainer as a motion teaser just because the user likes this travel energy
- treat a movement benchmark as proof of a structural film format

## Step 2 To Step 4 Calibration

What this case can influence:

- hero motion concept
- transition mechanics
- chase camera language
- spatial reveal pacing
- object-led rhythm

What this case should not directly dictate:

- explainer section logic
- MG modularity
- product demo sequence planning
- information-led batch structures

## Recommended Use Rule

If a future project references this case:

- use it as a movement and energy benchmark
- transfer object-anchor, chase-camera, and transition lessons
- keep structure anchored in the actual project type

If the project is really an explainer or workflow piece:

- do not let this case replace the correct structural benchmark
- use it only as a motion overlay if the user explicitly wants that hybrid
