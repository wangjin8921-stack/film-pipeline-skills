# Super Rep MG Case

Use this file as a calibration case, not as a universal template.

This reference captures one real `MG Explainer / Information Short` pattern from the local benchmark video `0208_灵北super.mp4`.

## Why This Case Matters

This case is useful because it is:

- clearly `MG Explainer / Information Short`
- strongly workflow-led rather than mood-led
- enterprise and internal-adoption adjacent without losing MG identity
- rich enough to show Step 1 through Step 4 behavior

Use it when the user wants:

- a benchmark for MG-type classification
- a real sample for preventing type drift
- a workflow-led enterprise MG pattern
- calibration against a reference video

Do not use it as proof that all MG videos should look or flow this way.

## Core Case Summary

Working description:

- an enterprise AI explainers piece about `Super Rep`
- likely medical or pharma commercial support context
- centered on visit preparation, visit support, and post-visit learning
- expressed through flat illustration, UI screens, cards, icons, and modular labels

Approximate structure:

1. define the AI system and its role
2. explain pre-visit preparation
3. explain in-visit support
4. explain post-visit learning and AI coaching
5. close on ecosystem and future-value language

## Dominant Type Judgment

Primary type:

- `MG Explainer / Information Short`

Secondary supporting traits:

- `Internal Promo / Internal Adoption Short`
- `Enterprise Promo`

Not primary type:

- `Concept Narrative Short`
- `Product Promo Short`
- `Interview Film`
- `Warm-Up / Hype Video`
- `AI cinematic short film`

## Why It Is MG

Transferable MG signals in this case:

- information is the main payload
- graphic-composite logic dominates
- icons, cards, labels, and UI blocks carry meaning
- scenes illustrate workflow states rather than dramatic acting
- the piece is modular and section-led
- visual transitions connect concepts more than physical action

## What Is Specific To This Case

These are case-specific decisions, not MG rules:

- red and white enterprise palette
- flat corporate illustration style
- pharma or medical-commercial setting
- HCP and physician visit workflow
- phone UI as a primary interface object
- `pre-visit -> in-visit -> post-visit` business loop
- a polished enterprise tone with a future-facing closing slogan

Do not generalize these details to all MG work.

## What Must Not Be Overgeneralized

Do not conclude from this case that:

- every MG piece should use flat character illustration
- every MG piece needs a phone screen
- every MG piece needs healthcare workflow language
- every MG piece should use red-and-white branding
- every MG piece should follow the same five-section structure
- every MG piece should end with enterprise slogan language

## Reusable Guardrails

These lessons transfer well to future MG projects:

- keep the dominant lane information-led
- make the logic readable before adding flourish
- build a design system before building many shot variants
- use repeated module types instead of inventing every shot from scratch
- let workflow or concept progression drive the structure
- use people only as support carriers of information, not as the main dramatic engine

## Reusable Freshness Levers

These are safe places to vary future MG projects without leaving the type:

- information metaphor
- module or card grammar
- icon family
- chart or compare-state mechanics
- UI framing style
- section naming system
- transition logic
- workflow shape
- final framing or takeaway language

This means a future MG piece can feel very different while still staying inside the MG lane.

## Step 1 Calibration

What this case teaches Step 1:

- classify workflow-led enterprise explainers as `MG Explainer / Information Short` when information design is the dominant delivery system
- allow enterprise tone as a supporting layer without upgrading the type to `Enterprise Promo`
- treat people and office scenes as support context when they do not carry the core storytelling logic

What Step 1 should not do:

- mistake corporate polish for brand-film logic
- mistake illustrated people for narrative short-film logic
- mistake app UI presence for pure `UI Demo / Screenflow Video`

## Step 2 Calibration

What this case teaches Step 2:

- storyboard around a business or knowledge flow, not around faux-cinematic scene coverage
- use section logic such as `what it is -> before -> during -> after -> system vision`
- let repeated module families create clarity
- keep shots readable and purpose-specific

What Step 2 should not do:

- turn MG into live-action-style shot writing
- over-fragment into hype-style microbeats
- force emotional acting beats where the real value is process clarity

## Step 3 Calibration

What this case teaches Step 3:

- prefer `graphic-composite`
- write prompts around UI blocks, cards, labels, icons, comparison states, and workflow diagrams
- use human figures as explanatory context, not as prompt-heavy performance subjects
- preserve a modular design system across sections

What Step 3 should not do:

- write cinematic live-action prompts for an MG explainers piece
- overstuff each prompt with mood adjectives
- let every shot invent a new visual language

## Step 4 Calibration

What this case teaches Step 4:

- execute in this order: design system -> card or icon families -> UI family -> section variants -> final composites
- keep reusable modules separate from final outputs
- isolate text, UI, and chart assets from character or environment support art
- validate layout clarity before scaling output volume

What Step 4 should not do:

- begin with large motion batches before the design system is locked
- treat the project like character-first animation production
- mix all assets into one undifferentiated folder structure

## Recommended Use Rule

If a future project feels adjacent to this case:

- transfer the MG guardrails
- transfer only the relevant structural lesson
- choose fresh motif, palette, module style, and workflow framing for the new project

If a future project does not share the same business shape:

- do not force this case structure onto it
- use it only to remind the system what makes MG distinct from narrative or hype lanes
