# Execution Type Profiles

Use this file to adapt Step 4 output to the actual film format.

This file is about execution order, batch strategy, material organization, and recovery logic.

## How To Use These Profiles

Treat the listed execution focus, approval order, workflow bias, batch bias, folder bias, and avoid-lines as production guardrails.

Then choose freshness levers that can live inside the production plan without destabilizing continuity, clarity, or budget.

Good places to vary:

- one hero experiment batch
- title or graphic system flavor
- motif-specific support assets
- alternate reveal beats
- optional pickup patterns
- folder or naming details that preserve the project's visual logic

Do not let experimentation erase the type's production priorities.

## Execution Freshness Levers By Type Family

### Information-Led Types

Good freshness levers:

- alternate card families
- compare-panel variants
- icon pack flavor
- transition system experiments
- optional explanatory pickup modules

Keep locked:

- design system first
- readable labels
- clear state progression
- controlled composites

### People-Led Types

Good freshness levers:

- support-cutaway subfamilies
- environment proof variants
- evidence insert sets
- alternate chapter cards
- secondary emotional punctuation shots

Keep locked:

- anchor continuity first
- speaker or protagonist stability
- restrained motion
- support coverage discipline

### Hype-Led Types

Good freshness levers:

- hero impact batch
- alternate text-hit families
- repeatable rhythm swaps
- optional reveal pickups
- payoff beat variants

Keep locked:

- fast throughput
- cheap retryability
- high readability
- low dependency chains

### Trust-Led Types

Good freshness levers:

- proof-angle alternates
- environment detail pickups
- symbolic hero stills
- before-and-after support modules
- restrained title card variants

Keep locked:

- credibility-first approvals
- coherent tone
- clear proof sequence
- low gimmick density

### Narrative-Led Types

Good freshness levers:

- motif expansion batches
- transformation milestone alternates
- emotional transition pickups
- hero payoff variants
- atmosphere-support plates

Keep locked:

- core transformation sequence
- continuity-safe asset order
- clear escalation
- manageable production scope

## Product Promo Short

- Execution focus: product clarity first, payoff second
- Approve first: hero product anchor and one clean use-context frame
- Workflow bias: still-to-video for product beauty, selected direct-video for visible use moments
- Batch bias: same product, same surface, same use environment
- Folder bias: `product_anchor`, `use_cases`, `result_shots`
- Avoid: spending budget on abstract mood shots before the product look is locked

## Enterprise Promo

- Execution focus: trust, scale, and operational credibility
- Approve first: one brand-safe environment family and one people-at-work anchor family
- Workflow bias: mixed still-to-video and reference-assisted realism
- Batch bias: environment continuity, team continuity, process proof
- Folder bias: `brand_environments`, `team_selects`, `process_motion`
- Avoid: over-fragmenting into ad-like micro-batches

## Brand Image Film

- Execution focus: hero imagery and motif continuity
- Approve first: signature hero frame and color-language lock
- Workflow bias: hero stills first, then restrained animation passes
- Batch bias: motif family, palette family, hero reveal family
- Folder bias: `hero_frames`, `motif_variants`, `motion_finishes`
- Avoid: treating every shot like a separate world

## Internal Promo / Internal Adoption Short

- Execution focus: practical clarity and relatable workflow
- Approve first: workplace anchor, UI state anchor, one proof-of-use frame
- Workflow bias: still-to-video plus UI/composite support
- Batch bias: same office context, same UI state, same task family
- Folder bias: `workflow_anchor`, `ui_states`, `benefit_shots`
- Avoid: using expensive motion passes where a controlled still or composite explains better

## Opening Video

- Execution focus: reveal timing and stage-scale impact
- Approve first: hero reveal frame, title system, signature motif shot
- Workflow bias: hero stills, selected direct-video reveals, graphic-composite titles
- Batch bias: reveal beats, title beats, motif beats
- Folder bias: `hero_reveals`, `title_system`, `impact_motion`
- Avoid: diffusing the opening with too many utility shots

## Warm-Up / Hype Video

- Execution focus: momentum, speed, and repeatable impact
- Approve first: one rhythm template, one text-hit template, one hero impact look
- Workflow bias: short direct-video plus graphic-composite text support
- Batch bias: rhythm family, impact family, transition family
- Folder bias: `rhythm_hits`, `text_hits`, `hero_impacts`
- Avoid: continuity-heavy approval gates that slow volume production

## Event Teaser

- Execution focus: anticipation and withheld reveal
- Approve first: teaser motif, countdown style, one suspense reveal frame
- Workflow bias: reveal-oriented still-to-video plus composite assets
- Batch bias: countdown assets, slogan assets, hint shots
- Folder bias: `countdown`, `teaser_reveals`, `cta_cards`
- Avoid: resolving the visual mystery too early in execution

## Event Recap

- Execution focus: compress highlight energy without chaos
- Approve first: event look lock, stage family lock, audience family lock
- Workflow bias: direct-video for peaks, still-to-video for hero memories
- Batch bias: stage, audience, signage, reaction families
- Folder bias: `stage_highlights`, `audience_energy`, `event_branding`
- Avoid: trying to build exhaustive chronological coverage

## Same-Day Edit / Flash Recap

- Execution focus: speed and usable throughput
- Approve first: one grade/look lock and one title layer lock
- Workflow bias: short direct-video and lightweight composite support
- Batch bias: quick-repeat event peaks and logo/text hits
- Folder bias: `fast_selects`, `logo_hits`, `same_day_pickups`
- Avoid: fragile continuity systems that create bottlenecks

## MC / Guest Intro Video

- Execution focus: entrance framing and prestige clarity
- Approve first: hero portrait, name/title system, one impact reveal shot
- Workflow bias: hero stills plus controlled graphic-composite treatment
- Batch bias: intro layout family and reveal family
- Folder bias: `guest_hero`, `name_cards`, `intro_reveals`
- Avoid: inventing multiple unrelated visual systems for one subject

## Program Interstitial / Segment Bumper

- Execution focus: fast branded transition
- Approve first: bumper design system and timing-safe title variants
- Workflow bias: `graphic-composite`
- Batch bias: one bumper system, many title swaps
- Folder bias: `bumper_master`, `segment_titles`, `exports`
- Avoid: treating bumpers like narrative scenes

## Interview Film

- Execution focus: speaker stability and support evidence
- Approve first: interview master frame, secondary angle if needed, one cutaway family
- Workflow bias: hybrid-reference and still-to-video
- Batch bias: interview master, quote-support cutaways, evidence inserts
- Folder bias: `interview_master`, `cutaway_family`, `quote_support`
- Avoid: generating many decorative motion shots before the speaker look is locked

## Customer Testimonial

- Execution focus: believable proof and sincerity
- Approve first: testimonial speaker anchor and one proof environment family
- Workflow bias: hybrid-reference or still-to-video
- Batch bias: speaker proof, work environment proof, result proof
- Folder bias: `speaker_anchor`, `proof_environment`, `result_support`
- Avoid: generic b-roll batches that do not support the claim

## Case Story Short

- Execution focus: before/after clarity
- Approve first: before-state lock, after-state lock, compare graphic system
- Workflow bias: still-to-video plus compare-oriented composite work
- Batch bias: before family, intervention family, after family
- Folder bias: `before_state`, `intervention`, `after_state`, `compare_cards`
- Avoid: mixing before and after looks without a clean approval break

## Street Interview / Vox Pop

- Execution focus: variety with readable capture style
- Approve first: street backdrop family and question-card system
- Workflow bias: short direct-video or reference-assisted capture style
- Batch bias: answer family, reaction family, question-card family
- Folder bias: `street_answers`, `reaction_bits`, `question_cards`
- Avoid: over-polishing the footage into a corporate interview look

## Training / Process Explainer Short

- Execution focus: step clarity and process accuracy
- Approve first: process map, screen or card template, step-state naming system
- Workflow bias: `graphic-composite`, UI demo, controlled support motion
- Batch bias: step templates, repeated process cards, state changes
- Folder bias: `step_templates`, `screen_states`, `guided_outputs`
- Avoid: animating before the step sequence and screen states are approved

## Policy / Compliance Communication

- Execution focus: rule clarity and behavior guidance
- Approve first: warning-card system, rule-callout system, scenario template
- Workflow bias: graphic-composite plus simple scenario shots
- Batch bias: rule cards, do/don't scenarios, reminder end cards
- Folder bias: `rule_cards`, `scenario_examples`, `reminder_assets`
- Avoid: abstract cinematic batches that weaken instruction clarity

## Recruitment Promo

- Execution focus: human appeal and team credibility
- Approve first: team anchor family, environment family, mission text style
- Workflow bias: still-to-video and hybrid-reference
- Batch bias: team moments, work-context moments, value or culture support
- Folder bias: `team_anchor`, `work_context`, `mission_support`
- Avoid: relying on generic atmosphere shots without real work context

## Leadership Message Film

- Execution focus: message authority and credibility
- Approve first: leader address master, message-safe support cutaways, title card style
- Workflow bias: stable still-to-video and hybrid-reference
- Batch bias: main address, support cutaways, key-message cards
- Folder bias: `leader_master`, `message_support`, `message_cards`
- Avoid: flashy effect batches that compete with the speaker

## MG Explainer / Information Short

- Execution focus: layout system before shot volume
- Approve first: design system, card style, icon style, label hierarchy
- Workflow bias: `graphic-composite`
- Batch bias: design system first, shot variations second
- Folder bias: `design_system`, `cards`, `icons`, `output_variants`
- Avoid: trying to execute it like live-action coverage

## Data Visualization Film

- Execution focus: metric readability and modular charts
- Approve first: chart grammar, label grammar, number emphasis system
- Workflow bias: `graphic-composite`
- Batch bias: chart modules, counter modules, compare modules
- Folder bias: `chart_modules`, `counter_modules`, `compare_panels`
- Avoid: stuffing too many statistics into one asset family

## UI Demo / Screenflow Video

- Execution focus: screen-state control and action clarity
- Approve first: UI state map, zoom style, annotation style
- Workflow bias: `graphic-composite` and hybrid-reference
- Batch bias: screen family, zoom family, annotation family
- Folder bias: `screen_states`, `zoom_views`, `annotations`, `outputs`
- Avoid: generating motion before the exact UI states are organized

## Flash / Quick-Cut Montage

- Execution focus: speed, impact, and reusable rhythm
- Approve first: rhythm template family and text-hit family
- Workflow bias: short direct-video and graphic-composite text hits
- Batch bias: repeatable rhythm templates and swap-friendly overlays
- Folder bias: `quick_hits`, `text_punches`, `payoff_beats`
- Avoid: high-dependency shot chains that make pickups expensive

## Concept Narrative Short

- Execution focus: motif continuity and staged transformation
- Approve first: symbolic anchor frame, environment family, transformation milestone frames
- Workflow bias: still-to-video with strong hero keyframes
- Batch bias: motif continuity, environment continuity, transformation phases
- Folder bias: `motif_anchor`, `transformation_steps`, `hero_payoffs`
- Avoid: generating disconnected beautiful shots with no transformation structure

## Emotional Atmosphere Short

- Execution focus: mood progression and tonal consistency
- Approve first: palette lock, light-direction lock, emotional anchor frame
- Workflow bias: still-to-video
- Batch bias: palette family, light family, emotional motif family
- Folder bias: `mood_anchor`, `palette_variants`, `soft_motion`
- Avoid: cutting the plan into too many aggressive batches

## Trailer / Teaser Style Short

- Execution focus: anticipation, fragments, and reveal control
- Approve first: title reveal system, dramatic beat family, withheld-information hero frames
- Workflow bias: direct-video for dramatic beats, graphic-composite for titles
- Batch bias: reveal family, title family, tension family
- Folder bias: `dramatic_beats`, `title_reveals`, `withheld_clues`
- Avoid: linear execution that gives away the full narrative too early

## Documentary-Style Enterprise Film

- Execution focus: observational authenticity
- Approve first: real-space anchor, process anchor, speaker realism lock
- Workflow bias: hybrid-reference and stable still-to-video
- Batch bias: real spaces, process detail, testimonial support
- Folder bias: `real_space`, `process_detail`, `human_support`
- Avoid: glossy over-designed batches that break documentary tone

## Type Selection Rules

- Follow the normalized `video_type` from Step 1 and the prompt bias from Step 3.
- If a format is information-led, approve the design system before batch output.
- If a format is people-led, approve the main speaker or protagonist anchor before support assets.
- If a format is hype-led, reduce approval depth and increase batch repeatability.
- If a format is trust-led, slow down the first approval gate and keep later variations restrained.
