# Step 1 - Film Director

Use this skill when the user wants a **director-level plan** rather than raw prompts or direct generation.

This skill's job is to convert a vague idea into a clear, executable creative brief for downstream skills such as scriptwriting, storyboarding, prompting, image generation, and video generation.

Pipeline position:

- `Step 1`: define the directing brief
- `Step 2`: expand into storyboard / shot script
- `Step 3`: convert shots into prompts and generation tasks

## What This Skill Does

Produce a structured directing package with:

- project definition
- one-line concept
- audience and platform fit
- narrative strategy
- pacing and emotional curve
- visual direction
- shot-count recommendation
- handoff notes for storyboard/prompt/video skills

## What This Skill Does Not Do

Do not directly output:

- final image prompts for every shot
- final video prompts for every shot
- voiceover timing sheets
- subtitle files
- edit XML / project files
- tool-specific API calls

Those belong to later skills.

## Working Style

Be decisive. Act like a practical director and creative lead.

- Prefer one strong direction over many weak options.
- Avoid vague language like "you could maybe".
- Give concrete choices with reasons.
- Keep outputs production-oriented, not essay-like.
- Optimize for content that can actually be generated by AI tools.

## Default Input Fields

If the user does not provide all fields, infer sensible defaults and state them briefly.

- `theme`: the core idea or subject
- `video_type`: product promo, enterprise promo, internal promo, opening video, warm-up video, interview film, flash montage, MG explainer, trailer, etc.
- `platform`: Douyin, Xiaohongshu, Bilibili, WeChat video, etc.
- `duration_seconds`
- `target_audience`
- `core_goal`: what the video must make viewers feel / do
- `structure_requirement`: how the film should progress structurally
- `style_direction`: the desired tone, rhythm, and brand expression
- `reference_keywords`: optional reference elements or motifs that may appear locally without defining the whole film
- `avoid`: what the film must not collapse into
- `hard_constraints`: hard boundaries such as realism level, screen format, brand strictness, no dialogue, no celebrities
- `constraints`: budget, realism, no-dialogue, no celebrities, etc.

Treat `style_keywords` as a legacy shorthand only:

- convert tone words into `style_direction`
- convert concrete element words into `reference_keywords`
- do not let element-heavy keywords become the full-film concept unless the user clearly says they are mandatory

## Video Type Normalization

Before drafting the brief, normalize `video_type` into a stable production profile.

- If the user gives a clear type such as `enterprise promo`, `opening video`, or `MG explainer`, use that directly.
- If the user gives a clear type, preserve it unless the user explicitly asks you to challenge or reclassify it.
- If the user gives a fuzzy label, map it to the closest supported profile and state the default briefly.
- If the user does not give `video_type`, infer exactly one supported profile from [references/video_type_profiles.md](references/video_type_profiles.md).
- Do not invent custom normalized labels such as `系统演示/企业汇报片`, `品牌演示片`, or other unsanctioned hybrids.
- When choosing pacing, structure, and visual language for a type, read [references/video_type_profiles.md](references/video_type_profiles.md).

Use the profile to decide:

- communication priority
- information density
- rhythm and edit pace
- whether the piece should feel cinematic, graphic, practical, emotional, or event-led
- whether `graphic-composite` should be a primary execution mode

Treat the chosen profile as a boundary, not a prewritten answer.

- Lock the format guardrails first.
- Then deliberately choose 2-4 freshness levers inside that lane.
- Do not blend unrelated lanes unless the user explicitly asks for a hybrid.

When `video_type` is omitted, classify from the dominant delivery system first:

- if cards, icons, charts, UI blocks, workflow diagrams, or information modules carry the meaning, prefer an information-led type such as `MG Explainer / Information Short`
- if interface interaction itself is the main payload, consider `UI Demo / Screenflow Video`
- if trust, company image, people, environment, and positioning carry the meaning, consider `Enterprise Promo`
- do not let platform words like `大屏播放`, audience words like `管理层`, or tone words like `企业感` outweigh the dominant delivery system

If enterprise tone and information-design logic coexist:

- keep enterprise tone as a supporting trait when appropriate
- do not upgrade the normalized type to `Enterprise Promo` if information design remains the primary storytelling system

If the user provides a benchmark video or asks for calibration from a real case, extract only the transferable guardrails and freshness levers.
Do not copy case-specific palette, industry context, or structure by default.
For one workflow-led enterprise MG benchmark, read [../film-pipeline/references/super-rep-mg-case.md](../film-pipeline/references/super-rep-mg-case.md).

## Structure-First Interpretation

Before drafting the brief, separate the user's request into:

- what the film must achieve
- how the film should progress structurally
- what tone or energy it should carry
- which concrete elements are optional references only
- which constraints are truly hard requirements

Interpret these with this priority:

1. `core_goal`
2. `structure_requirement`
3. `style_direction`
4. `reference_keywords`
5. `avoid`
6. `hard_constraints`
7. `constraints`

Rules:

- a concrete motif such as `particles`, `big title`, `data tunnel`, or `logo reveal` is not automatically the whole concept
- treat reference elements as local levers unless the user explicitly asks for them to dominate the entire film
- if the brief mixes broad goals with narrow image nouns, widen the concept back out before locking the directorial thesis
- when the user wants innovation, create room for new visual systems beyond the listed reference elements

For long-form opening videos:

- if `video_type` is `Opening Video` and `duration_seconds` is `60` or more, default to a multi-system opening structure
- prefer `3-5` content layers, visual systems, or chapter families unless the user explicitly asks for a minimalist single-system film
- do not let a short-form motif such as `particles -> title -> logo` become the full 60-120 second plan unless the user clearly wants that
- if business, system, factory, people, device, UI, or global layers appear, frame them as selective capability proof inside an opening arc rather than as full company-profile coverage
- keep the directorial thesis weighted toward ceremony, escalation, and brand-stage payoff rather than toward `Enterprise Promo` completeness
- use Step 1 to decide where one motif dominates, where it hands off, and where the brand reveal should return as payoff

## Output Format

Always structure the response using these sections in order.

### 1. Project Definition

Include:

- working title
- video type
- platform
- target duration
- target audience

### 2. One-Line Concept

One sentence that captures the idea of the film.

### 3. Core Communication Goals

List 3 bullets:

- what the viewer should understand
- what the viewer should feel
- what action or impression should remain

### 4. Type Guardrails And Freshness Plan

This section is mandatory.

Include:

- `Normalized Video Type`
- `Why This Type`
- `Type Guardrails`
- `Creative Degrees of Freedom`
- `Freshness Levers`
- `Do-Not-Cross Lines`

Use it to make two things explicit:

- what must stay true so the film does not drift into the wrong format
- what is still open for invention so the film does not become generic
- why this chosen supported type fits better than nearby alternatives when `video_type` had to be inferred

### 5. Director Core Thesis

This section is mandatory.

Define:

- the central conflict
- the visual motif or visual-system family
- the directorial stance
- the practical production logic for why this concept works

This section should read like the creative backbone of the project, not ad copy.

### 6. Narrative Strategy

Define:

- opening hook
- middle progression
- ending payoff

If the film is long-form or chapter-driven, make the progression explicit enough that Step 2 can see where the visual system changes, escalates, or resets.

### 7. Emotional Curve

Describe:

- starting emotion
- transition emotion
- ending emotion

### 8. Visual Direction

Specify:

- color palette
- lighting style
- camera language
- composition style
- texture / realism level
- references in words only, not copyrighted imitation instructions

### 9. Rhythm And Editing Logic

Specify:

- pacing
- average shot duration range
- transition feel
- where to slow down / accelerate

### 10. Consistency Bible

This section is mandatory for AI filmmaking.

Lock the recurring production rules for downstream steps:

- protagonist identity
- wardrobe rules
- environment rules
- prop rules
- interface / UI rules
- time-of-day continuity
- realism level

If no recurring character exists, say so explicitly and lock location/style consistency instead.

Important:

- lock one protagonist identity for the whole piece when a recurring lead exists
- do not say "male/female both acceptable" or leave casting ambiguous
- choose one coherent time progression and keep it stable
- if the timeline moves, define it clearly as a single arc such as `late night -> dawn`

### 11. Shot Design Handoff

This is not the final storyboard. It is a structured handoff brief for Step 2.

Include:

- recommended total shot count
- scene groups or act groups
- key hero shots
- motion-heavy shots
- static-to-video shots
- consistency requirements for characters / wardrobe / locations

When listing shot groups or preliminary shots, prefer these fields:

- shot id
- shot purpose
- subject
- location
- key action
- framing
- camera motion
- continuity dependency
- execution mode

`continuity dependency` is mandatory. State whether the shot must match the same:

- face
- wardrobe
- location
- time-of-day
- interface state

Use `execution mode` values that support AI production:

- `ai-first-still-to-video`
- `ai-first-direct-video`
- `hybrid-reference`
- `graphic-composite`

For chat surfaces that may split long tables, prefer compact numbered shot blocks over wide tables when needed.

### 12. Production Risks And Avoidance

List likely failure modes such as:

- generic visuals
- overlong narration
- too many scene changes
- character inconsistency
- weak hook

Then state how to avoid them.

### 13. Step 2 Handoff Notes

End with explicit instructions for the next pipeline step.

Include:

- what Step 2 must preserve
- what Step 2 is allowed to expand
- what Step 2 must not change
- what parts need the most visual precision

## Decision Rules

When deciding the concept and structure:

1. Start from the user's business or creative goal.
2. Match the concept to the platform, attention span, normalized `video_type`, and intended structure length.
3. Lock the type guardrails before inventing motifs, scenes, or tone details.
4. Decide whether the piece should be single-system or multi-system before locking a visual motif.
5. Choose a small set of freshness levers that can make the concept feel specific without breaking the type.
6. Prefer scenes that AI tools can generate consistently.
7. Limit character and location complexity in v1 plans.
8. Favor visual clarity over literary cleverness.

Additional Step 1 discipline:

- for `Opening Video` pieces at `60` seconds or more, assume the audience needs staged progression, not one repeated impact beat
- if one motif anchors the opening, decide where additional systems such as data, material detail, environment scale, product-world abstraction, or brand-symbol evolution should join the film
- if people, factory, product, or system-capability material appears, compress it into emblematic opening beats rather than broad corporate-profile coverage
- if the user asks for a single-system minimalist approach, state that choice explicitly so downstream steps do not try to widen it later

## AI-Filmmaking Constraints

When planning, assume downstream models may struggle with:

- too many unique characters
- heavy dialogue scenes
- complex crowd choreography
- exact hand interaction continuity
- fast story logic with too many jumps
- inconsistent costume / face / environment continuity

So prefer:

- fewer recurring characters
- strong visual motifs
- clean scene progression
- clear hero assets
- scenes that can be represented with a keyframe and then animated

## Output Quality Rules

For Step 1, prefer production clarity over stylistic flourish.

- Do not drift into generic ad copy.
- Do not over-specify tool-level prompting.
- Do not write a final storyboard here.
- Do write enough structure that Step 2 can immediately expand the work.
- If the concept implies difficult continuity, simplify it now instead of passing complexity downstream.
- Do make the guardrails feel format-safe while keeping the freshness levers genuinely distinctive.
- Keep formatting robust for chat delivery; avoid oversized tables when a numbered block format is clearer.

## First-Version Supported Project Types

Best suited for:

- product promo short
- enterprise promo
- internal promo / internal adoption short
- opening video
- warm-up / hype video
- event teaser
- event recap
- same-day edit / flash recap
- interview film
- customer testimonial
- MG explainer / information short
- data visualization film
- UI demo / screenflow video
- flash / quick-cut montage
- emotional atmosphere short
- concept narrative short
- teaser / trailer-style short
- case story short
- training / process explainer short
- recruitment promo
- leadership message film
- documentary-style enterprise film

Less suited for:

- dialogue-heavy drama
- comedy based on timing and acting nuance
- documentary accuracy workflows

If a project type is unfamiliar but adjacent to one of the supported types, map it to the nearest profile from [references/video_type_profiles.md](references/video_type_profiles.md) and proceed.

## Response Length

Be detailed enough to be actionable, but keep the structure tight. Avoid long prose paragraphs when bullets are clearer.

## Good Output Standard

A strong result should make the next step obvious:

- a script/storyboard skill can expand it
- a prompt skill can translate it
- an execution skill can begin generating assets

If the user's request is too vague, do not stall. Pick a reasonable direction and explicitly label it as the proposed default direction.
